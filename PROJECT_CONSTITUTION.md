# TrayMotors プロジェクト憲法

## 何を作るか

TrayMotors は、CPU・メモリ・GPU の状態を、Windows の通知領域にいる3体の小さい機械キャラで見せるツール。

狙いは「かわいい監視ツール」ではなく、PCの負荷を一瞬で読める軽い常駐ツールにすること。監視対象のPCを、監視アプリ自身が重くしてはいけない。

## 中核の仕組み

このアプリが成立するかは、次の2つが信頼できるかで決まる。

- 使用率が上がるほど、動きが速くなる
- 温度や圧迫度が上がるほど、色が危険寄りになる

この対応が分かりづらいなら、キャラを増やしても意味がない。

## 見た目のルール

- CPU は 2気筒ピストン。
- メモリはタービン。メモリ圧迫時は、速くなるだけではなく、回転が不安定になったり遅くなったりする。
- GPU はロータリーエンジン。
- 通知領域サイズで識別できることを最優先にする。細かい装飾は後。
- 詳細UIは必要なときだけ開く。v1では常時表示ダッシュボード、WebView、アニメーション窓は作らない。

## 実装のルール

- v1はWindows専用。
- C# / .NET 10 / WinForms `NotifyIcon` で作る。
- 計測、状態、アニメーションは分ける。
- CPU/GPU/メモリ使用率は PDH と Win32 を軸にする。
- LibreHardwareMonitor は温度用。任意ONで、失敗してもアプリは落とさない。
- アイコンは事前生成してキャッシュする。毎tickで新しい画像を描かない。
- 取れない値はグレー表示とソース表示で逃がす。嘘の数値やクラッシュより、その方がまし。

## v1でやらないこと

- 履歴グラフは作らない。
- テーマエディタは作らない。
- NVIDIA/AMD/Intel専用SDKには寄せない。
- インストーラ、自動更新、コード署名はまだやらない。
- Windows 11のタスクバー本体へ無理に埋め込まない。

## 品質ライン

- アイドル時CPU使用率は、普通のタスクマネージャー上で気にならないレベルにする。
- メモリ使用量は、常駐トレイツールとして小さく保つ。
- CPU使用率はTask Managerに近づける。RunCat 365で報告されていたような極端な過小表示は避ける。
- GPUカウンターや温度センサーが取れなくても、起動失敗にしない。
